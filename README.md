# Coding-lab_Group30

## KNH Digital Infrastructure Project

This project was completed by Group 30 as part of the Group Coding Lab assignment.

The goal of the project is to help Kenyatta National Hospital (KNH) manage and analyze data generated from critical hospital sensors. Using shell scripting, Linux permissions, log analysis, and Git collaboration, our team built tools to initialize the system, secure data, analyze sensor readings, and archive logs safely.

## Project Files

### hospital_system.py

The hospital monitoring engine provided for the assignment.

This script continuously generates sample data for:

- Heart Rate sensors
- Temperature sensors
- Water Usage sensors

Start the engine:

bash python3 hospital_system.py start 

Stop the engine:

bash python3 hospital_system.py stop 

---

### hospital_admin.sh

This script prepares and secures the working environment.

#### initialize_system()

Created by Marie Nishimwe.

Responsibilities:

- Check whether required directories exist

Directories managed:

- active_logs
- archived_logs
- reports

#### secure_data()

Created by Mireille.

Responsibilities:

- Restrict access to active log files

#### Execution Logic

Created by Chol.

- Execute initialization and security functions

### hospital_analysis.sh

This script analyzes data generated by the monitoring engine.

#### process_vitals()

Created by Friend.

Responsibilities:

- Search Heart Rate and Temperature logs

#### water_audit()

Created by Friend.

Responsibilities:

- Analyze ICU water usage records

### hospital_archive.sh

Created by Germain.

Responsibilities:

- Move processed logs from active_logs to archived_logs
- Rename archived logs using timestamps

Example:

## Security and Privacy

To comply with KNH data handling requirements, generated logs and reports are excluded from version control.

The following entries are included in the .gitignore file:

text active_logs/ archived_logs/ reports/ /tmp/hospital_system.pid 

This ensures that generated operational data is not uploaded to GitHub.

## How to Run the Project

Start the monitoring engine:

bash python3 hospital_system.py start 

Initialize and secure the environment:

bash bash hospital_admin.sh 

Analyze generated data:

bash bash hospital_analysis.sh 

Archive processed logs:

bash bash hospital_archive.sh 

Stop the monitoring engine:

bash python3 hospital_system.py stop
